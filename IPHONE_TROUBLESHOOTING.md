# 📱 iPhoneアクセス問題のトラブルシューティング

## 🔍 確認事項

### 1. ブラウザキャッシュをクリア
iPhoneのSafariで：
1. 設定 → Safari → 履歴とWebサイトデータを消去
2. または、Safariで「共有」→「ホーム画面に追加」を削除して再追加

### 2. URLを確認
- ✅ 正しいURL: `https://vmeda.vercel.app`
- ❌ 間違ったURL: `http://vmeda.vercel.app` (HTTPS必須)

### 3. ネットワーク接続を確認
- Wi-Fiまたはモバイルデータが有効か確認
- 他のサイトにアクセスできるか確認

### 4. Vercelのデプロイ状況を確認
1. https://vercel.com/dashboard にアクセス
2. プロジェクト `vmeda` を選択
3. 「Deployments」タブで最新のデプロイが成功しているか確認
4. エラーがある場合は、ログを確認

### 5. サーバーログを確認
Vercelダッシュボードで：
1. プロジェクト → 「Functions」タブ
2. 最新のログを確認
3. iPhoneからのアクセス（User-Agentに`iPhone`が含まれる）を確認

## 🛠️ よくある問題と解決方法

### 問題1: 白い画面が表示される
**原因**: JavaScriptが読み込まれていない
**解決方法**:
- ブラウザの開発者ツール（SafariのWebインスペクタ）でエラーを確認
- `app.js`と`styles.css`が正しく読み込まれているか確認

### 問題2: 404エラーが表示される
**原因**: 静的ファイルが見つからない
**解決方法**:
- Vercelのデプロイが完了しているか確認
- 数分待ってから再アクセス

### 問題3: CSPエラーが表示される
**原因**: Content Security Policyの設定
**解決方法**:
- サーバーログでCSPエラーを確認
- 必要に応じてCSP設定を調整

### 問題4: ネットワークエラー
**原因**: ネットワーク接続の問題
**解決方法**:
- Wi-Fiとモバイルデータを切り替えて試す
- 他のサイトにアクセスできるか確認

## 📊 デバッグ方法

### SafariのWebインスペクタを使用
1. iPhoneの設定 → Safari → 詳細 → Webインスペクタを有効化
2. MacのSafariで「開発」→ iPhone名 → サイト名を選択
3. コンソールでエラーを確認

### サーバーログを確認
Vercelダッシュボードで：
- 「Functions」タブ → 最新のログ
- iPhoneからのアクセスを確認（User-Agentに`iPhone`が含まれる）
- エラーメッセージを確認

## 🎯 次のステップ

1. **Vercelのデプロイが完了しているか確認**
   - ダッシュボードで最新のデプロイが成功しているか確認

2. **iPhoneで再アクセス**
   - ブラウザキャッシュをクリア
   - `https://vmeda.vercel.app` にアクセス

3. **エラーが続く場合**
   - Vercelのログを確認
   - サーバーログでiPhoneからのアクセスを確認
   - エラーメッセージを共有してください
